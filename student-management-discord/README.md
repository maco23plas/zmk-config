# 生徒管理シート × Discord 連携ツール

「AIC_商品開発管理シート」（Instagram運用代行スクールの生徒管理）を Discord と連携し、
**リマインドの自動送信・改善提案・ダッシュボード化**を行う Google Apps Script です。

シートに何も追加でインストールする必要はありません。スプレッドシートに最初から付いている
「Apps Script」機能だけで動きます。

---

## できること

**配信タイミング：毎週 月曜・金曜 8:00（JST）に、指定チャンネルへ「リマインド＋状況報告」をまとめて送信。**

### 1. 要対応リマインド（講師メンション付き）
対応が必要な生徒を自動で検知し、**講師ごとにまとめ＋@メンションで本人を呼び出して** Discord に通知します。

| 種類 | 条件 |
|------|------|
| 🔴 音信不通 | ステータスが「音信不通」 |
| 🟠 週報未提出 | KPI報告事項に「2週以上週報提出なし」 |
| 🟠 稼働低下 | KPI報告事項に「稼働時間_基準以下」 |
| 🟡 フォロー | ステータスが「経過観察」「問題あり」 |
| 🎓 卒業間近 | 卒業日まで 30/14/7/3/1 日 |
| 🔄 休会復帰 | 休会終了日まで 7/3/1 日 |
| ⚠️ KPI期限超過 | ステージ別KPIの期限を過ぎても未達（後述） |

### 2. 状況報告（サマリー＋改善提案）
在籍数・KPI達成・収益・ステータス／ステージ／講師別の内訳に加え、
- 音信不通の再接触リスト
- 昇格・称号UP候補（実績がステージを上回っている生徒）
- KPI期限超過の未達リスト
- 卒業間近×成果ゼロの生徒
- 講師別の要対応負荷

を自動でまとめて提案します。

### 3. シートを再デザイン（🎨 コーポレート・ブルー）
メニュー **🎨 シートを見やすく整形** で、生徒管理タブの見た目を一括で刷新します。
**データの値は一切変更せず、見た目だけ**を整えます（配色・配置は再実行で何度でも適用可）。

- 見出し行＝ロイヤルブルー×白文字（行固定）
- 生徒行＝白／極薄ブルーの1行おきストライプ
- 月区切り行（2025/06 など）＝ネイビーのセクションバー
- ステータス＝はっきり信号色（問題無=緑／経過観察=黄／問題あり=橙／音信不通=赤）
- ステージ＝アドバンス＝青系・マスター＝紫系 など
- KPI報告事項（週報未提出など）を赤系でハイライト、🚨アラート列を重大度で色分け
- 「基本情報｜進捗・ステージ｜休会｜KPI｜収益｜メモ」を縦の仕切り線でグループ化
- 収益は「¥1,234」表示、説明・報告事項は折り返し、列幅を最適化

> 配色は `Code.gs` 冒頭の `THEME` で変更できます。
> 同じタブに上部・下部の集計ブロックが同居しているため、列の挿入・並べ替えは行いません
> （他ブロックの崩れを防ぐため）。整形は生徒テーブルの範囲だけに適用されます。

### 4. シート内「🚨アラート」列の自動更新
各生徒の行に、要対応の内容を絵文字付きで表示。シートを開くだけで状況が一目でわかります。

### 5. 「📊ダッシュボード」シートの自動生成
集計指標をまとめた専用タブを自動作成・更新します。

---

### 6. 管理サイト（ウェブアプリ）🌐
スプレッドシートのデータをそのまま読み取り、**スマホでも開けるモダンな管理サイト**として表示します。
ダッシュボード（KPIカード・内訳グラフ・改善提案）／生徒一覧（検索・絞り込み・カード→詳細）／
要対応（講師別）を備え、画面のボタンから **Discordのリマインド・状況報告をワンタップ送信**できます。
（データの値は変更しません。閲覧＋通知が中心の v1 です）

**公開手順（デプロイが必要）:**
1. Apps Scriptエディタで `Index.html`（ファイル → 新規 → HTML）を作り、リポジトリの `Index.html` を貼り付け
2. 右上 **デプロイ → 新しいデプロイ → 種類: ウェブアプリ**
3. 「次のユーザーとして実行: 自分」「アクセス: 自分のみ（または組織内）」を選び **デプロイ**
4. 表示された **ウェブアプリURL** を開く（スマホはホーム画面に追加するとアプリ感覚）

> コードを更新したら **デプロイ → デプロイを管理 → 編集（鉛筆）→ バージョン: 新バージョン → デプロイ** で反映。

## セットアップ（5分）

### STEP 1. スクリプトを貼り付け
1. 対象スプレッドシートを開く
2. メニュー **拡張機能 → Apps Script**
3. 既存コードを消して、本リポジトリの `Code.gs` の中身を**全部**貼り付け
4. 💾 で保存 → スプレッドシートを再読み込み

すると上部に **「📋生徒管理ツール」** メニューが追加されます。

### STEP 2. Discord Webhook を作成（＝送信先チャンネルの指定）
1. **通知を送りたいチャンネル**を開く（このチャンネルが送信先になります）
2. **チャンネル設定（⚙️）→ 連携サービス → ウェブフック → 新しいウェブフック**
3. 名前（例: 生徒管理Bot）を付けて **ウェブフックURLをコピー**

### STEP 3. URLを登録して接続テスト
1. スプレッドシートのメニュー **📋生徒管理ツール → ① Discord Webhookを設定**
2. コピーしたURLを貼り付け
3. **🧪 Discord接続テスト** を実行 → Discordに「✅接続テスト成功」が届けばOK
   - 初回のみGoogleの承認画面が出ます。「許可」を選んでください。

### STEP 4. 講師メンションを設定（@で本人を呼ぶ）
`Code.gs` 冒頭の `CONFIG.instructorMentions` に、各講師のDiscord IDを入れます。

```js
instructorMentions: {
  '砂川': '<@111111111111111111>',   // ← 砂川さん本人をメンション（ユーザーID）
  '上野': '<@&222222222222222222>',  // ← 「上野チーム」ロールをメンション（ロールID）
  '土方': '333333333333333333',      // ← 数字IDだけでもOK（ユーザー扱い）
},
```

**IDの調べ方**：Discordの「設定 → 詳細設定 → 開発者モード」をON →
ユーザー（またはロール）を右クリック →「ユーザーIDをコピー」/「ロールIDをコピー」。
- ユーザーをメンション： `<@ID>` または 数字IDだけ
- ロールをメンション： `<@&ID>`
- 空のままだと、その講師は名前表示のみ（ピングなし）になります。

### STEP 5. 自動配信をON
**📋生徒管理ツール → ② 自動配信をON（月・金 8:00）**
- 毎週 **月曜・金曜 8:00** に、アラート列／ダッシュボード更新 → リマインド（講師メンション付き）→ 状況報告 をまとめて送信します。

（手動送信：「🚀 今すぐ リマインド＋状況報告を送信」／個別に「🔔 リマインドのみ」「📊 状況報告のみ」も可能）

> 曜日・時刻を変えたい場合は `CONFIG.scheduleWeekDays`（例 `['MONDAY','FRIDAY']`）と
> `CONFIG.scheduleHour`（例 `8`）を編集し、再度「② 自動配信をON」を実行してください。

### STEP 6.（任意）シートを整形
**📋生徒管理ツール → 🎨 シートを見やすく整形** を一度実行すると色分け・固定・列幅が整います。
条件付き書式なので、以降は手動で実行しなくても色は自動で追従します。

## カスタマイズ（`Code.gs` 冒頭の `CONFIG`）

```js
graduationReminderDays: [30, 14, 7, 3, 1],   // 卒業の何日前に通知するか
leaveReturnReminderDays: [7, 3, 1],          // 休会復帰の何日前に通知するか

kpi: {                                        // ステージ別KPI（基準の経過日数・売上）
  'ゼロイチ':   { days: 30, revenue: 1,      label: '初案件（トライアル）獲得' },
  'アドバンス': { days: 60, revenue: 100000, label: '10万円以上の案件獲得' },
  'マスター':   { days: 60, revenue: 150000, label: '15万円以上の案件獲得' },
},

instructorWebhooks: {        // 講師ごとに別チャンネルへ送りたい場合だけURLを入れる
  '砂川': '', '上野': '', '土方': '',   // 空ならメインチャンネルに集約
},
```

### 講師ごとにチャンネルを分けたい場合
講師専用チャンネルでそれぞれ Webhook を作り、`instructorWebhooks` に貼るだけ。
各講師には自分の担当生徒のリマインドだけが届きます。

## 仕組みのメモ

- **シート・列の自動検出**：「名前／ステータス／講師／ステージ」を含む見出し行を探し、
  列は**名前で対応付け**するため、列の並びが多少変わっても動きます。
  どうしても見つからない場合は `CONFIG.studentSheetName` にシート名を直接指定してください。
- **日付**：`MM/DD` 形式・`YYYY/MM/DD` 形式・Google日付セルのいずれにも対応。
  卒業日／休会終了日は「今後の予定」として、入会日／進級日は「過去」として年を推定します。
- **KPI判定**：`最大収益` がステージ基準に達していれば達成。`KPI対象外=TRUE` や休会中の生徒は除外。
- **Webhook URL の保管**：コードには書かず、スクリプトプロパティに安全に保存します。
